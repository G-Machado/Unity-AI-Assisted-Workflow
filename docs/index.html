The ritual

Every task follows the same sequence — no shortcuts, no skipped steps.

🔍
/investigate <task>

Investigate

Loads project docs for context. Root cause analysis, affected files, 3 approaches proposed. No code written.

✅

You approve one approach

Choose Quick Fix, Optimal Fix, or Balanced Fix. Claude waits — nothing happens without your explicit sign-off.

⚙️
/implement

Implement stop after

Branch created (with your confirmation). Minimum necessary changes. Self-review checklist. Stops before commit.

🧪
/validate

Validate in Unity Editor manual

You open the Editor. Claude guides compilation check, Play Mode testing, MCP inspection. Routes to fix-error on failure.

🔧
/fix-error <error>

Fix errors (if any)

Checks ERRORS.md for known solutions first. Classifies error, proposes 3 fixes, logs the resolution for next time.

📦
/commit

Commit + update docs no push

Logical atomic commits, Conventional Commits format, plan shown before execution. Updates project docs automatically. You push manually.

Installation

Run this from your Unity project root. Works on Windows (Git Bash), macOS, and Linux.

1

Run the one-liner from your Unity project root

# cd into your Unity project first
cd "C:/Users/you/Projects/MyGame"

bash <(curl -fsSL https://raw.githubusercontent.com/G-Machado/Unity-AI-Assisted-Workflow/main/install.sh)

The script validates your Unity project, downloads the package, auto-detects your Unity Editor path, and generates .mcp.json.

2

Open Claude Code in the project

claude

Uses your Pro or Max subscription — no API key needed. Just log in with your claude.ai account.

3

Bootstrap project documentation

/doc-init

Scans your existing scripts and generates the docs layer. One-time setup — updates automatically after that.

4

Start your first task

/investigate add double-jump to the player controller

Requirements: Claude Code CLI, Unity 2021.3+, Node.js (for unity-mcp-server). Git Bash on Windows.

Living documentation

Four compact files in .claude/docs/ — structured data, never prose. Auto-updated by the workflow so Claude always has full context without wasting tokens re-discovering the project.

PROJECT.md

Current branch, open work, known issues, next planned task.

Updated by → /commit
SYSTEMS.md

All scripts ever touched: path, type, dependencies, serialized fields.

Updated by → /commit
DECISIONS.md

Why X was chosen over Y — prevents re-litigating architecture in every session.

Updated by → /commit
ERRORS.md

Every error resolved, with root cause and fix. Checked before new analysis begins.

Updated by → /fix-error

Commands

/investigate <task> Root cause analysis, 3 approaches, no code written. Reads docs for context. /implement Branch + minimum changes after approved approach. Stops before commit. /validate [error] Guide Unity Editor validation. Routes to fix-error or commit on result. /fix-error <error> Classify, propose 3 fixes, check ERRORS.md for known solutions. Logs resolution. /commit Atomic commits with plan approval. Updates docs automatically. Never pushes. /doc-init Scan project and generate .claude/docs/ from scratch. /doc-update Manually refresh docs after an out-of-band change. /ritual-init Bootstrap .claude/ and .mcp.json for a new project (for manual installs).

Trade-offs

This package is opinionated by design. Know what you're getting.

Strengths

  • You stay in control — Claude proposes, you decide
  • No surprise commits, no autonomous pushes — ever
  • Docs auto-update so context never degrades between sessions
  • Known errors are remembered — same mistake never analyzed twice
  • Past decisions are logged — Claude won't re-argue settled choices
  • Zero dependencies — pure Claude Code primitives, no runtime
  • Works with your existing subscription — no extra API cost
  • One command installs into any Unity project on any machine

Constraints

  • Slower than letting AI act autonomously — every step requires approval
  • Unity MCP inspection requires npm and a running Editor instance
  • Git Bash required on Windows (native CMD not supported)
  • Docs must be initialized manually with /doc-init on first use
  • Does not auto-run tests — manual Play Mode validation is required
  • Subagents (investigator, implementer, validator) count toward usage

The 7 laws

Loaded every session via CLAUDE.md. Non-negotiable.
